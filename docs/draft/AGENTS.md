# AGENTS.md Draft

本文件是面向 `openrelay` 仓库的 `AGENTS.md` 收敛草案。

目标是把 `AGENTS.md` 收敛成一个稳定、低噪音的入口文件：只保留少量全局约束，并把 repo 特有信息按需外挂，而不是常驻在默认上下文中。

除非用户在当前任务中明确覆盖，否则所有改动都应遵循以下原则。

## 1. 文件定位

`AGENTS.md` 只承担三类职责：

- 说明仓库的默认协作方式
- 给出少量稳定、全局、必须遵守的约束
- 说明在什么情况下应读取哪一类外挂文档

`AGENTS.md` 默认不承载以下内容：

- 过长的流程说明
- 只在部分任务中生效的详细门槛
- 大量 repo 事实、模块细节、背景材料
- 示例过多的写作规范

这类信息应放到独立文档，并且只在命中触发条件时读取。

## 2. 默认协作方式

AI 默认先判断当前请求是否已经足够清晰。

- 如果目标、边界、约束或验收明显不清楚，先澄清问题或收敛方案，再决定是否实现
- 如果任务明确、局部、低风险，可以直接进入实现；但仍应先用一句话复述任务理解
- 如果任务涉及设计、架构、兼容性或跨文件行为修改，默认先检查是否需要读取对应外挂协议

默认目标不是“尽快开始写”，而是在不增加无谓流程负担的前提下，避免把模糊请求直接翻译成结构债。

## 3. 全局约束

### 提交要求

- 每次完成一轮可独立成立的改动后，都应进行一次 `git commit`
- 提交粒度应尽量聚焦：一个提交只解决一个明确问题，避免把不相关改动混在一起
- 提交信息应准确描述“为什么改”以及“改了什么”，避免空泛标题
- 如果任务天然包含多个独立部分，应拆成多个语义清晰的提交，而不是堆成一个大提交
- 如果改动尚未通过最基本的自检，不应急于提交

### 实施要求

- 改动前先理解现有结构，再下手修改
- 优先做根因修复，不做表面 patch
- 尽量避免“顺手修很多地方”；除非确有必要，否则保持改动聚焦
- 在不影响清晰度的前提下，保持实现简洁
- 复杂改动应先想清楚边界、状态和数据流，再动手

### 结构与兼容性要求

- 主路径应尽量短、直、稳定，不应被边角兼容逻辑淹没
- 相近逻辑应被收敛，而不是复制出多个相似版本
- 默认不为了“可能有人在用”而保留旧接口、旧分支、旧数据路径
- 如果确实需要兼容，必须有明确理由，并控制兼容层范围
- 能删除的旧代码，不要保留“以后可能有用”的残留实现

### 文档要求

- 影响使用方式、配置方式、架构分层的改动，应同步更新文档
- 新增设计决策、取舍理由、未决问题时，优先记录到 `docs/design/`
- 如果任务主状态依赖设计任务板，应按任务板协议回写证据和状态

## 4. 外挂文档读取规则

只有命中触发条件时，才主动读取外挂文档；没有命中时，不默认扩展读取。

### 4.1 需求澄清与进入实现条件

命中以下任一情况时，先读取 `docs/draft/ai-specification-gate.md`：

- 用户请求的目标、边界、约束或验收不清楚
- 任务涉及新功能、重构、架构调整、兼容性调整
- 改动会影响跨文件行为、外部接口、状态流转或配置方式
- 当前无法判断应直接实现，还是应先做澄清或方案分析

以下情况通常不必先读取该文档：

- 明确且局部的 bug 修复
- 文案调整
- 小型测试补充
- 边界和验收都已清楚的局部配置修改

### 4.2 设计任务板与任务状态更新

命中以下任一情况时，先读取 `docs/design/task-board-protocol.md`：

- 当前任务对应 `docs/design/open-questions.md` 中的明确任务条目
- 本轮改动完成后，需要判断任务是否可以关闭
- 需要回写任务证据、follow-up 或状态

如果当前任务没有绑定设计任务板条目，或本轮不涉及任务状态更新，则不必主动读取该协议。

## 5. 外挂信息分层

repo 特有信息默认按下面三层外挂，而不是直接堆回 `AGENTS.md`：

- `协议`：定义什么时候该澄清、什么时候该计划、什么时候可以实施
- `任务板规则`：定义任务如何进入、如何关闭、如何回写证据
- `设计与模块文档`：记录具体设计决策、边界、历史取舍和局部事实

只有“稳定、全局、几乎每轮都会影响行为”的规则，才应该提升回 `AGENTS.md`。

## 6. 反膨胀约束

如果后续继续演化本文件，默认遵守以下收敛规则：

- 不把 repo 背景资料、模块说明、长示例直接并入主文件
- 不把仅对少数任务生效的决策树写成全局默认规则
- 不把外挂协议中的细节复制回主文件，只保留触发条件和索引
- 如果新增内容无法回答“为什么必须常驻默认上下文”，就不应进入 `AGENTS.md`

## 7. 当前外挂索引

- `docs/draft/ai-specification-gate.md`：需求澄清、阶段判断、进入实现前的最低条件
- `docs/design/task-board-protocol.md`：设计任务的关闭条件、证据回写、自勾选规则

如果后续新增 repo 特有文档，也应优先继续沿用“主入口最小化、信息按需外挂”的结构，而不是重新回到一个超长的总规文件。

如果任务与上述原则冲突，以用户当前明确要求为准。
