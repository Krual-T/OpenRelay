# Requirements

## Goal
围绕“当前这条会话正在做什么、停在哪里、下一步能做什么”收敛一个稳定入口。

## Problem Statement
停止、状态查看、继续等待态、压缩上下文等动作虽然基本可用，但入口仍分散；飞书线程里缺少稳定的当前会话控制面。

## Required Outcomes
1. 完成正式设计稿，明确当前会话状态模型、控制入口位置、高频动作集合和低频动作下沉策略。
2. 代码主路径里至少提供停止、查看状态、继续等待态、压缩上下文四类高频动作的集中入口。
3. 飞书端可稳定区分运行中、等待用户、已完成、失败、已停止等核心状态。

## Non-Goals
- 不在 package migration 这一步直接实现功能代码。
- 不在需求未澄清前提前透支更复杂的交互或产品面。

## Constraints
- 当前 package 是从历史遗留任务描述提炼出的正式设计入口。
- 后续实现前仍应补充更细的总体设计和详细设计。
