# OR-TASK-008 工作区目录选择卡设计

更新时间：2026-03-18

## 背景

当前工作区切换主路径依赖 `/cwd <path>` 和 `/cd <path>`。这条路径有三个问题：

- 用户必须知道目录结构，且要手输路径，交互成本高。
- 产品入口仍然偏 CLI 心智，不像飞书里的用户产品。
- 目录切换语义和面板语义脱节，目录卡片只是 `/cwd` 的按钮外壳，不是真正的目录选择体验。

这轮要把“切目录”从命令式输入收敛为卡片式选择，把 `/cwd`、`/cd` 从用户主路径移除。

## 目标

- 提供一个面向飞书用户的工作区选择器，而不是命令提示。
- 以单一 workspace root 为边界，支持逐级浏览目录，而不是只停在首层。
- 工作区根在卡片里统一显示为 `~`，避免把真实物理根目录名直接暴露成产品概念。
- 默认从 `~/Projects` 打开，但允许通过环境变量调整。
- 卡片交互应支持原地刷新、分页、返回上一级、搜索、隐藏目录显式开关和快捷入口复用。

## 不做什么

- 这轮不做任意深度的文件树浏览。
- 这轮不做搜索框或表单输入。
- 这轮不继续保留 `/cwd`、`/cd` 作为用户文档主路径。

## 交互模型

### 1. 主入口

- `/workspace` 打开工作区选择卡。
- `/panel workspace` 作为面板内的对应结果面。
- `/help` 和 `/panel` 里的相关按钮都跳转到 `/workspace`。

### 2. 卡片结构

工作区选择卡分成三段：

1. 当前状态段
   - 当前会话目录
   - 当前浏览目录
   - 说明“点目录进入下一层，点选中按钮切换当前 scope”
2. 搜索与导航段
   - 输入框搜索当前浏览目录下的子目录
   - `选中当前目录`
   - `返回上一级`
   - `回到根目录`
   - `显示隐藏目录 / 隐藏目录`
3. 子目录列表段
   - 只展示当前浏览目录下的直接子目录
   - 每行目录用 `interactive_container` 承载，点击即进入下一层
   - 分页条采用 `/resume` 类似的页码条 + 上一页/下一页

### 3. 交互容器与输入框语义

- 目录列表采用 Feishu Card JSON 2.0 的 `interactive_container` 承载每一行目录入口。
- 整行可点击，不要求用户点小按钮。
- 行内同时承载标题、相对路径和当前态。
- 搜索采用表单输入框，提交后把 `form_value` 合并进 `/workspace --query ...` 命令。
- callback value 统一回传命令和上下文。

参考文档：

- https://open.feishu.cn/document/feishu-cards/card-json-v2-components/containers/interactive-container

## 状态与边界

- 工作区根目录来自 `WORKSPACE_ROOT` / `WORKSPACE_DIR`，默认值改为 `~/Projects`。
- 产品文案里把该根目录统一显示为 `~`，避免用户看到“Projects 是顶层”的错误心智。
- 目录浏览和目录选择都只允许在该 root 内解析，不允许逃逸。
- 目录列表默认过滤 `.` 开头目录，避免把 `.git`、`.venv` 等内部目录直接暴露给用户。
- 当用户显式开启“显示隐藏目录”时，当前浏览目录下的隐藏子目录也应进入列表，并在翻页、搜索、返回时保持该开关状态。
- 切换目录会清空当前 scope 的 backend thread 与上下文占用，下一条真实消息在新目录重新启动。

## 为什么改成逐级浏览

- 用户明确需要返回上一层，而不仅是分页。
- 只做首层目录会把“目录树”压扁成一次性列表，不适合真实项目结构。
- 继续要求用户手输任意路径，又会退回 `/cwd` 的命令式心智。

因此这轮收敛为：

- 单一根目录内的逐级浏览
- 当前目录下子目录搜索
- `/resume` 风格分页
- 快捷目录复用相同切换语义

## 已落第二阶段实现

- 新增 `/workspace` 主入口。
- `/panel` 增加 `workspace` 结果面。
- 工作区根默认改为 `~/Projects`，并通过环境变量配置。
- 工作区卡现在支持逐级浏览、返回上一级、回到根目录、搜索和分页。
- 工作区卡新增“显示隐藏目录”开关，默认仍隐藏 `.` 开头目录。
- 快捷目录按钮复用 `/workspace select <path>`，展示路径改为 `~` 根别名语义。
- `/cwd`、`/cd` 从帮助文案、README 和主面板语义中移除。
